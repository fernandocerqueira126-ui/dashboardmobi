# 03-03-PLAN.md — Refinamento da Proposta e UI

## 1. Mapeamento
- **App.tsx**: Adicionar a rota `/proposta` (opcionalmente sem ID).
- **Sidebar.tsx**: Adicionar novo item `Criar Proposta` sob `Atendimentos` na área de Gestão, com ícone DocumentText/FileText.
- **LeadsCRM.tsx**: Refatorar a visualização dos botões do `LeadDetailSheet` para Grid 2x3. Cores usando hexadecimais extraídos de `LeadsContext.tsx`.
- **Proposta.tsx**:
    - Suporte a seleção manual de Lead quando `id` for nulo ou alterar o atual.
    - Ocultar aviso da falta de chave da IA, usando `import.meta.env.VITE_GEMINI_API_KEY` apenas na lógica de request.
    - Buscar propriedades reais utilizando a tabela `imobiliaria_rag`.

## 2. Abordagem (Passo a Passo)
1. **Routing**: Modificar `App.tsx` para permitir rota sem o parâmetro.
2. **Sidebar**: Alterar `Sidebar.tsx`, importar ícone novo e registrar a rota `/proposta`.
3. **LeadsCRM.tsx**: Substituir a seção de botões de Ação. Configurar com grid de 2 colunas.
4. **Proposta.tsx**: Reescrever o RAG para buscar na base autêntica. Implementar select box do lead. Eliminar UI de erro do Gemini.

## 3. Validação
- O Layout de Lead Detail deve comportar perfeitamente 2 colunas.
- A rota `/proposta` deve carregar sem quebrar, mostrando o select.
- A busca RAG deve retornar itens verdadeiros do banco de dados e aplicar ao contexto de prompt.
- `VITE_GEMINI_API_KEY` deve trabalhar escondida.
